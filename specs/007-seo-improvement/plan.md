# Implementation Plan: SEO Implementation Improvement

**Branch**: `007-seo-improvement` | **Date**: 2026-02-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/007-seo-improvement/spec.md`

---

## Summary

Consolidate the split SEO architecture (full `Seo` component on Landing only; minimal `SEOHead` on all other pages) into a single, centralized pipeline where `TemplateLayout` uses the existing full-featured `Seo` component from `packages/ui` for every public page. Extend `SEOConfig` with Open Graph, Twitter Card, canonical URL, and robots fields. Populate all routes with complete metadata. Add `WebPage`, `BreadcrumbList`, `WebSite+SearchAction`, and `FAQPage` structured data schemas via a new schema generator utility. Add `<lastmod>` to sitemap output. Remove all per-page `<Seo>` calls and delete the now-superseded `SEOHead` component.

---

*Goal: Establish the type contracts that all subsequent phases depend on. No runtime behavior changes yet.*

**1.1 — Extend `OpenGraph` type in `packages/ui/src/components/Seo/Seo.tsx`**

Add `imageWidth?: number`, `imageHeight?: number`, `imageType?: string` to the `OpenGraph` interface. Update the `Seo` component render logic to emit:

### Phase 2: Schema Generator Utility
*Goal: Create the reusable utility that generates `WebPage` + `BreadcrumbList` schemas for all routes.*

**2.1 — Create `apps/web/src/utils/schema-generators.ts`**

```typescript
const BASE_URL = 'https://modularhouse.ie';

export function generatePageSchema(
  path: string,
  title: string,
  description: string,
  buildTimestamp: string
): SchemaDef[] {
  const fullUrl = path === '/' ? `${BASE_URL}/` : `${BASE_URL}${path}`;
  const isHome = path === '/';

  const breadcrumbItems = isHome
    ? [{ '@type': 'ListItem', position: 1, name: 'Home' }]
    : [
        { '@type': 'ListItem', position: 1, name: 'Home', item: BASE_URL },
        { '@type': 'ListItem', position: 2, name: title },
      ];

  return [
    {
      type: 'WebPage',
      data: {
        '@id': `${fullUrl}#webpage`,
        url: fullUrl,
        name: title,
        description,
        datePublished: buildTimestamp,
        dateModified: buildTimestamp,
        breadcrumb: { '@id': `${fullUrl}#breadcrumb` },
      },
    },
    {
      type: 'BreadcrumbList',
      data: {
        '@id': `${fullUrl}#breadcrumb`,
        itemListElement: breadcrumbItems,
      },
    },
  ];
}

export function generateWebSiteSchema(): SchemaDef {
  return {
    type: 'WebSite',
    data: {
      '@id': `${BASE_URL}/#website`,
      url: `${BASE_URL}/`,
      name: 'Modular House',
      description: 'Steel Frame Garden Rooms & House Extensions',
      potentialAction: [{
        '@type': 'SearchAction',
        target: {
          '@type': 'EntryPoint',
          urlTemplate: `${BASE_URL}/?s={search_term_string}`,
        },
        'query-input': {
          '@type': 'PropertyValueSpecification',
          valueRequired: true,
          valueName: 'search_term_string',
        },
      }],
    },
  };
}
```

**2.2 — Write unit tests: `apps/web/src/utils/__tests__/schema-generators.test.ts`**

Test cases:
- `generatePageSchema('/', ...)` returns exactly 2 SchemaDefs; breadcrumb has 1 list item; `@id` ends in `#webpage`
- `generatePageSchema('/garden-room', ...)` returns breadcrumb with 2 list items; item[0] has `item: 'https://modularhouse.ie'`
- `generatePageSchema` embeds the passed `buildTimestamp` in both `datePublished` and `dateModified`
- `generateWebSiteSchema()` returns a SchemaDef with type `'WebSite'` and a `potentialAction` array with one `SearchAction`

Target: 100% branch coverage.

### Phase 4: TemplateLayout Consolidation
*Goal: Single SEO component injection point. Eliminates the architectural split.*

**4.1 — Update `apps/web/src/components/TemplateLayout.tsx`**

Replace `SEOHead` with the full `Seo` component:

```tsx
// REMOVE:
import { SEOHead } from './seo/SEOHead';

// ADD:
import { Seo } from '@modular-house/ui';
```

Replace the `SEOHead` render call:
```tsx
// REMOVE:
{currentRoute?.seo && (
  <SEOHead config={currentRoute.seo} titleSuffix=" | Modular House" />
)}

// ADD:
{currentRoute?.seo && (
  <Seo
    title={currentRoute.seo.title}
    description={currentRoute.seo.description}
    canonicalUrl={currentRoute.seo.canonicalUrl}
    robots={currentRoute.seo.robots}
    openGraph={currentRoute.seo.openGraph}
    twitter={currentRoute.seo.twitter}
    jsonLd={currentRoute.seo.schema ? {
      '@context': 'https://schema.org',
      '@graph': currentRoute.seo.schema.map(s => ({ '@type': s.type, ...s.data }))
    } : undefined}
    siteTitleSuffix=" | Modular House"
  />
)}
```

**Important**: The existing `StructuredData` component was used inside `SEOHead` to handle `schema?: SchemaDef[]`. With the consolidation, schemas are passed to `Seo` as `jsonLd` using the `@graph` pattern (matching the competitor's Yoast approach). The `StructuredData` component is no longer needed by `TemplateLayout` — but leave it in place as it may be useful for future one-off schema injection needs.

Accept an optional `buildTimestamp` parameter. Add `<lastmod>` to each `<url>` block:
```typescript
export function generateSitemapXml(buildTimestamp?: string): string {
  const lastmod = buildTimestamp
    ? buildTimestamp.split('T')[0]  // YYYY-MM-DD
    : new Date().toISOString().split('T')[0];

  const urlElements = routes.map(route => `
  <url>
    <loc>${HOSTNAME}${route.path}</loc>
    <lastmod>${lastmod}</lastmod>
    <changefreq>${route.sitemap?.changefreq || 'monthly'}</changefreq>
    <priority>${route.sitemap?.priority || 0.5}</priority>
  </url>`).join('');
  // ...
}
```
